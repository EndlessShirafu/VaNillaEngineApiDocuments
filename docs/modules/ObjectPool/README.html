ObjectPool

Object の実体を保持するプールです。 本モジュールは ObjectFactory と協調動作することを前提に設計してありますが、単体でも動作します。

概要

ObjectPoolは、VaNillaエンジンにおけるオブジェクトの効率的な管理のためのプーリングシステムです。オブジェクトの頻繁な生成・削除によるガベージコレクション圧を軽減し、パフォーマンスを向上させます。

主要機能

  • オブジェクト再利用: 一度生成されたオブジェクトをプールして再利用
  • メモリ効率: ガベージコレクションの負荷を軽減
  • 型安全: ジェネリクスによる型安全なオブジェクトプール
  • 自動拡張: プール不足時の自動オブジェクト生成
  • 統計情報: アクティブオブジェクト数の追跡

基本的な使用方法

// オブジェクトプールの作成
var pool = new ObjectPool<MyObject>(100); // 100個のMyObjectをプール

// オブジェクトの取得
MyObject obj = pool.AllocateObject();

// オブジェクトの使用
// ... オブジェクトを使用 ...

// オブジェクトの返却
pool.DeallocateObject(obj);

// デアロケート処理の実行（通常はフレーム終了時）
pool.ExecuteDeallocation();

API Reference

詳細なAPIドキュメントについては、VaNillaのAPIリファレンスを参照してください。

ObjectPool

指定型のVaNillaオブジェクトをプーリングする機構です。

コンストラクタ

  • ObjectPool(int objectPoolSize = 2000) - 指定サイズでオブジェクトプールを初期化

メソッド

  • AllocateObject() - プールからオブジェクトを取得（プールが空の場合は新規生成）
  • DeallocateObject(in ObjectType obj) - オブジェクトをプールに返却予約
  • ExecuteDeallocation() - デアロケート待ちのオブジェクトを実際にプールに返却
  • GetAllocatedObjectNum() - 現在アクティブなオブジェクト数を取得

ObjectPoolCache

指定型のオブジェクトプールをキャッシュする機構です。

静的プロパティ

  • PoolInstance - オブジェクトプールのキャッシュインスタンス

使用上の注意

  • DeallocateObjectは即座にプールに戻すのではなく、返却予約をします
  • 実際の返却処理はExecuteDeallocationを呼び出したときに実行されます
  • プールサイズは動的に拡張されますが、初期サイズを適切に設定することでパフォーマンスが向上します

ObjectFactory連携

ObjectPoolはObjectFactoryと組み合わせて使用することで、より高度なオブジェクト管理が可能になります。
