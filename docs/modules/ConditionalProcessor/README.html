ConditionalProcessor

オブジェクトの条件付き実行と自動消滅機能を提供するExpresserComponentモジュールです。ゲーム開発における一般的なライフサイクル管理パターンを実装します。

概要

ConditionalProcessorは、VaNillaエンジンにおけるオブジェクトの条件付き動作を管理するコンポーネント群です。時間制御、位置制御、カスタム条件による自動処理を提供し、特にシューティングゲームやアクションゲームでの弾丸、エフェクト、一時的なオブジェクトの管理に威力を発揮します。

主要コンポーネント

CountDownSelfDestruction

時間経過による自動消滅機能を提供します。

// 5秒後に自動消滅するエフェクト
var effect = ObjectActivator.Activate<EffectObject>(new List<IExpresserComponent> {
    new CountDownSelfDestruction { DurationSecond = 5.0f }
});

// 残り時間の確認
var countdown = effect.GetComponent<CountDownSelfDestruction>();
float remainingTime = countdown.RestTime;

特徴:

  • DurationSecond: 消滅までの時間（秒）
  • RestTime: 残り時間（読み取り専用）
  • 時間切れ時にObjectFactory.Terminate()を自動実行
  • 一時的なエフェクト、パワーアップアイテム、攻撃判定などに最適

ScreenOutSelfDestruction

画面外への移動による自動消滅機能を提供します。

// 画面外に出たら消滅する弾丸
var bullet = ObjectActivator.Activate<BulletObject>(new List<IExpresserComponent> {
    new ScreenOutSelfDestruction 
    { 
        Padding = (20, 20),           // 余白ピクセル
        WorkInsideScreen = true       // 一度画面内に入ってからチェック開始
    }
});

設定パラメータ:

  • Padding: 画面境界からの余白（X, Y）ピクセル
  • WorkInsideScreen: true = 一度画面内に入ってから機能開始
  • screenSizeManager: カスタム画面サイズ管理（省略時は自動取得）

動作モード:

  • 通常モード: オブジェクト生成直後から画面外チェック
  • 画面内モード: オブジェクトが一度画面内に入ってからチェック開始
  • 異常検出: 長時間画面外にある場合の警告機能

依存関係:

  • HasPositionコンポーネントが必須

DesiredConditionalProcessor

単一の条件-処理ペアによるカスタム動作を提供します。

var enemy = ObjectActivator.Activate<EnemyObject>(new List<IExpresserComponent> {
    new DesiredConditionalProcessor()
});

var processor = enemy.GetComponent<DesiredConditionalProcessor>();

// プレイヤーが近くにいたら攻撃
processor.desiredConditionalFunc = (obj, time) => 
{
    var playerDistance = CalculateDistanceToPlayer(obj);
    return playerDistance < 100.0f;
};

processor.desiredProcessorFunc = (obj, time) => 
{
    AttackPlayer(obj);
    return true; // 継続実行
};

デリゲート仕様:

  • desiredConditionalFunc: 条件判定関数（nullの場合は常に実行）
  • desiredProcessorFunc: 実行処理関数（false返却で自動消滅）

DesiredConditionalProcessors（複数形）

複数の条件-処理ペアを管理します。

var complexObject = ObjectActivator.Activate<ComplexObject>(new List<IExpresserComponent> {
    new DesiredConditionalProcessors()
});

var processors = complexObject.GetComponent<DesiredConditionalProcessors>();

// 複数の条件を順次チェック
processors.AddDesiredConditionalProcessor(
    (obj, time) => obj.Health <= 0,                    // 体力0以下
    (obj, time) => { PlayDeathEffect(); return false; } // 死亡処理
);

processors.AddDesiredConditionalProcessor(
    (obj, time) => time % 60 == 0,                     // 1秒毎
    (obj, time) => { RegenerateHealth(); return true; } // 体力回復
);

高度な機能:

  • FunctionAlternativeParameter: オブジェクト参照の置き換え
  • 順次実行: 条件がfalseを返すまで続行
  • パフォーマンス最適化: CollectionsMarshal.AsSpan使用

DesiredExecution / DesiredExecutions

条件なしのカスタム実行機能を提供します。

// 単一実行
var simpleObj = ObjectActivator.Activate<SimpleObject>(new List<IExpresserComponent> {
    new DesiredExecution()
});
simpleObj.GetComponent<DesiredExecution>().desiredExecutionFunc = 
    (obj, time) => { UpdateAnimation(); return true; };

// 複数実行
var multiObj = ObjectActivator.Activate<MultiObject>(new List<IExpresserComponent> {
    new DesiredExecutions()
});
var executions = multiObj.GetComponent<DesiredExecutions>();
executions.AddDesiredExecution((obj, time) => { UpdatePhysics(); return true; });
executions.AddDesiredExecution((obj, time) => { UpdateAI(); return true; });

実用的な使用例

弾幕シューティングゲーム

public class BulletFactory
{
    public static BulletObject CreatePlayerBullet(Point2 position, Vector2 velocity)
    {
        var bullet = ObjectFactory.Instantiate<BulletObject>();
        bullet.GetComponent<HasPosition>().Position = position;
        bullet.GetComponent<HasVelocity>().Velocity = velocity;
        
        // 画面外で自動消滅
        bullet.AddComponent(new ScreenOutSelfDestruction 
        { 
            Padding = (10, 10),
            WorkInsideScreen = false 
        });
        
        // 10秒で強制消滅（安全機能）
        bullet.AddComponent(new CountDownSelfDestruction 
        { 
            DurationSecond = 10.0f 
        });
        
        return bullet;
    }
    
    public static BulletObject CreateEnemyHomingBullet(Point2 position)
    {
        var bullet = ObjectFactory.Instantiate<BulletObject>();
        bullet.GetComponent<HasPosition>().Position = position;
        
        // 追尾機能
        bullet.AddComponent(new DesiredConditionalProcessor());
        var homing = bullet.GetComponent<DesiredConditionalProcessor>();
        
        homing.desiredConditionalFunc = (obj, time) => 
        {
            var player = FindPlayer();
            return player != null && Vector2.Distance(obj.Position, player.Position) > 50.0f;
        };
        
        homing.desiredProcessorFunc = (obj, time) => 
        {
            var player = FindPlayer();
            var direction = Vector2.Normalize(player.Position - obj.Position);
            obj.GetComponent<HasVelocity>().Velocity = direction * 300.0f;
            return true;
        };
        
        return bullet;
    }
}

パーティクルシステム

public class ParticleSystem
{
    public static void CreateExplosion(Point2 center)
    {
        for (int i = 0; i < 20; i++)
        {
            var particle = ObjectFactory.Instantiate<ParticleObject>();
            particle.GetComponent<HasPosition>().Position = center;
            
            // ランダム方向に飛散
            float angle = (float)(Random.NextDouble() * Math.PI * 2);
            float speed = (float)(Random.NextDouble() * 200 + 100);
            particle.GetComponent<HasVelocity>().Velocity = new Vector2(
                (float)Math.Cos(angle) * speed,
                (float)Math.Sin(angle) * speed
            );
            
            // 2-4秒でフェードアウト
            float lifetime = (float)(Random.NextDouble() * 2 + 2);
            particle.AddComponent(new CountDownSelfDestruction 
            { 
                DurationSecond = lifetime 
            });
            
            // 時間経過でアルファ値減少
            particle.AddComponent(new DesiredExecution());
            var fade = particle.GetComponent<DesiredExecution>();
            fade.desiredExecutionFunc = (obj, time) => 
            {
                var countdown = obj.GetComponent<CountDownSelfDestruction>();
                float alpha = countdown.RestTime / lifetime;
                obj.GetComponent<ChangeAlpha>().Alpha = alpha;
                return true;
            };
        }
    }
}

条件付きAIエネミー

public class SmartEnemy : Object
{
    public override void SetUp()
    {
        // 複数のAI行動パターン
        this.AddComponent(new DesiredConditionalProcessors());
        var ai = this.GetComponent<DesiredConditionalProcessors>();
        
        // 1. プレイヤー追跡
        ai.AddDesiredConditionalProcessor(
            (obj, time) => IsPlayerInRange(obj, 200.0f),
            (obj, time) => { ChasePlayer(obj); return true; }
        );
        
        // 2. 攻撃行動
        ai.AddDesiredConditionalProcessor(
            (obj, time) => IsPlayerInRange(obj, 100.0f) && CanAttack(obj),
            (obj, time) => { AttackPlayer(obj); return true; }
        );
        
        // 3. 体力低下時の逃避行動
        ai.AddDesiredConditionalProcessor(
            (obj, time) => obj.GetComponent<HasHitPoint>().HitPoint < 30,
            (obj, time) => { FleeFromPlayer(obj); return true; }
        );
        
        // 4. 体力0で死亡
        ai.AddDesiredConditionalProcessor(
            (obj, time) => obj.GetComponent<HasHitPoint>().HitPoint <= 0,
            (obj, time) => { PlayDeathAnimation(obj); return false; } // false = 自動消滅
        );
    }
    
    private bool IsPlayerInRange(Object obj, float range)
    {
        var player = FindPlayer();
        if (player == null) return false;
        
        var distance = Vector2.Distance(
            obj.GetComponent<HasPosition>().Position,
            player.GetComponent<HasPosition>().Position
        );
        return distance <= range;
    }
}

パフォーマンス考慮事項

最適化のポイント

  • 条件判定の軽量化: 重い計算は条件内で避ける
  • 適切な更新頻度: 毎フレーム不要な処理は時間間隔を設ける
  • 早期リターン: 条件が満たされない場合の高速終了
// 良い例: 軽量な条件判定
processor.desiredConditionalFunc = (obj, time) => 
{
    // 軽量なチェックを先に
    if (obj.GetComponent<HasHitPoint>().HitPoint <= 0) return true;
    
    // 重い計算は最後に
    if (time % 30 == 0) // 30フレームに1回だけ
    {
        return ComplexConditionCheck(obj);
    }
    return false;
};

// 悪い例: 重い処理を毎フレーム実行
processor.desiredConditionalFunc = (obj, time) => 
{
    // 毎フレーム重い計算を実行（避けるべき）
    return ComplexConditionCheck(obj);
};

メモリ効率

  • デリゲートのキャッシュを推奨
  • ラムダ式でのクロージャキャプチャに注意
// 推奨: デリゲートのキャッシュ
private static readonly DesiredConditionalFunc healthCheckFunc = 
    (obj, time) => obj.GetComponent<HasHitPoint>().HitPoint <= 0;

processor.desiredConditionalFunc = healthCheckFunc;

// 注意: クロージャによるメモリリーク
float capturedValue = 100.0f;
processor.desiredConditionalFunc = (obj, time) => 
{
    // capturedValueをキャプチャするためGCプレッシャー
    return obj.Health < capturedValue;
};

デバッグとトラブルシューティング

よくある問題と解決法

  1. オブジェクトが消滅しない

    // HasPositionコンポーネントが不足
var bullet = ObjectActivator.Activate<BulletObject>();
bullet.AddComponent(new ScreenOutSelfDestruction()); // エラー: HasPosition必須

// 解決法
bullet.AddComponent(new HasPosition());
bullet.AddComponent(new ScreenOutSelfDestruction());

  2. 条件が期待通りに動作しない

    // デバッグ用ログ追加
processor.desiredConditionalFunc = (obj, time) => 
{
    bool condition = SomeCondition(obj);
    VaNilla.Debug.Logger.LogInfo($"Condition result: {condition}");
    return condition;
};

  3. パフォーマンス問題

    // フレーム間隔での実行
private int frameCounter = 0;
processor.desiredConditionalFunc = (obj, time) => 
{
    frameCounter++;
    if (frameCounter % 10 == 0) // 10フレームに1回
    {
        return ExpensiveCondition(obj);
    }
    return false;
};

アーキテクチャ統合

ExpresserComponentとしての実装

  • 全てのプロセッサはIExpresserComponentを実装
  • SetUp/TearDown/Executeライフサイクルに対応
  • Activeプロパティによる有効/無効制御

ObjectFactoryとの連携

  • ObjectFactory.Terminate()との自動連携
  • オブジェクトプールへの適切な返却
  • メモリリークの防止

TimeManagerとの統合

  • フレーム時間との同期
  • 正確な時間計算
  • ポーズ機能への対応

依存関係

  • Transform: HasPosition（ScreenOutSelfDestruction用）
  • TimeManager: 時間計算
  • ScreenResolutionManager: 画面サイズ取得
  • ObjectFactory: オブジェクト消滅処理
  • DebugLogger: ログ出力

まとめ

ConditionalProcessorは、VaNillaエンジンにおけるオブジェクトライフサイクル管理の基盤となるモジュールです。時間制御、位置制御、カスタム条件による柔軟な動作制御を提供し、特にゲーム開発での一般的なパターンを効率的に実装できます。適切な使用により、保守性が高く、パフォーマンスに優れたゲームロジックを構築できます。
