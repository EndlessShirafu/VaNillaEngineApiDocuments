ObjectExecutor

VaNilla.Object の定期更新処理を行うためのモジュールです。
VaNillaエンジンにおいて、本モジュールへ登録されたVaNilla.Objectのみが
実際にゲームループ内で更新されます。

本モジュールはstaticクラスです。

APIリファレンス

詳細なAPIドキュメントはVaNilla ObjectExecutor APIリファレンスをご覧ください。

VaNilla.Objectを登録するには

ObjectExecutor.Registerメソッドを利用します。
ObjectExecutor.Registerを使って登録されたVaNilla.Objectはゲームループ内で定期的に更新されます。

ObjectExecutor.RegisterによるVaNilla.Objectの登録時に、内部でVaNilla.ObjectのSetUpおよび、紐づいている各ExpresserComponentのSetUpが呼ばれます。

注意点

  • ObjectFactory.Instantiate を利用して複製したVaNilla.Objectの場合、ObjectFactory.Instantiate の実行完了時点で自動的にObjectExecutorに登録されます。

  • ObjectActivator.Activate を利用して生成したVaNilla.Object の場合、ObjectActivator.Activate の実行完了時点で自動的にObjectExecutorに登録されます。

  • 上記の場合、 ObjectExecutor.Register を手動で呼び出してしまうと二重登録となり、不具合の原因になるため注意してください。

VaNilla.Objectを登録解除するには

ObjectExecutor.Unregisterメソッドを利用します。
ObjectExecutor.UnregisterされたVaNilla.Objectはゲームループ内で更新されることが無くなります。

ObjectExecutor.UnregisterによるVaNilla.Objectの登録解除時に、内部でVaNilla.ObjectのTearDownおよび、紐づいている各ExpresserComponentのTearDownが呼ばれます。

注意点

  • ObjectFactory.Instantiateを利用して複製したVaNilla.Objectの場合、ObjectExecutor.Unregisterの代わりに必ず ObjectFactory.Terminateを呼び出すようにしてください。
    ObjectFactory.Terminate 内部でObjectExecutor.Unregisterを呼び出すため、自動的にObjectExecutorから登録解除されます。
    そのため、ObjectFactory経由で複製したVaNilla.Objectについては、明示的にUnregisterを呼び出す必要はありません。
    (呼び出してもよいですが、意味はありません)
  • ObjectActivator.Activate を利用して生成したVaNilla.Object の場合、ObjectExecutor.Unregisterの代わりに必ず ObjectActivator.Deactivateを呼び出すようにしてください。
    • こちらも、内部でObjectExecutor.Unregisterを呼び出すため、自動的にObjectExecutorから登録解除されます。
