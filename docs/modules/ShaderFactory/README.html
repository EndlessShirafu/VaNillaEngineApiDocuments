ShaderFactory

シェーダーリソースの管理とレンダリング効果を提供するVaNillaモジュールです。 DxLibとUnity両方のプラットフォームで、統一されたシェーダー管理システムを実現します。

概要

ShaderFactoryは、2Dスプライトレンダリングのためのシェーダー管理機能を提供します。 加算合成、乗算合成、スクリーン合成などの一般的なブレンドモードをサポートし、 プラットフォーム固有の実装を抽象化して統一されたAPIを提供します。

APIリファレンス

詳細なAPIドキュメントはVaNilla ShaderFactory APIリファレンスをご覧ください。

主要機能

シェーダー管理

  • ShaderManager: 統一されたシェーダーリソース管理
  • ShaderResourceLoader: 非同期シェーダーローディング
  • ChangeShading: オブジェクト単位のシェーダー切り替え

サポートされるブレンドモード

ブレンドモード 説明 用途
DEFAULT 通常の描画 基本的なスプライト描画
ALPHA アルファブレンド 透明度を考慮した描画
ADD 加算合成 光や炎のエフェクト
SUB 減算合成 影や暗化エフェクト
MUL 乗算合成 影や色の変調
SCREEN スクリーン合成 明るい部分の合成
INVSRC 反転ソース 特殊エフェクト

基本的な使用方法

シェーダーの取得と設定

using VaNilla.Shader;

// デフォルトシェーダーの取得
var defaultShader = ShaderManager.GetDefaultShader();

// 特定のシェーダーの取得（Enum指定）
var addShader = ShaderManager.GetShader(ShaderManager.ShaderEnum.ADD);

// 特定のシェーダーの取得（文字列指定）
var multiplyShader = ShaderManager.GetShader("Multiply");

オブジェクトにシェーダーを適用

// ChangeShading コンポーネントを持つオブジェクトに対して
var changeShading = myObject.GetComponent<ChangeShading>();
changeShading.UpdateShader(ShaderManager.GetShader(ShaderManager.ShaderEnum.ADD));

カスタムシェーダーの登録

// 独自のシェーダーリソースを登録
var customShader = ShaderResourceLoader.Load("MyCustomShader");
ShaderManager.Register("MyCustomShader", customShader);

// 登録したシェーダーの使用
var myShader = ShaderManager.GetShader("MyCustomShader");

アーキテクチャ

3層構造

  1. Interface層 (Interface/): プラットフォーム非依存のAPI
  2. Implementation層 (Implementation/): プラットフォーム固有の実装
    • DxLib/: DxLib用の実装
    • Unity/: Unity用の実装
  3. Fixture層 (Fixture/): Unityプラットフォーム用のシェーダーアセット

リソース管理

  • 非同期ローディング: ShaderResourceLoaderによる効率的なリソース管理
  • 自動キャッシュ: 一度ロードしたシェーダーは自動的にキャッシュ
  • メモリ管理: 明示的なUnload APIによるメモリ解放

Unity専用シェーダー

Fixture/Unity/フォルダには、Unity用の標準スプライトシェーダーバリエーションが含まれています：

  • Sprites-Default.shader: 標準描画
  • Sprites-Alpha.shader: アルファブレンド
  • Sprites-Add.shader: 加算合成
  • Sprites-Subtract.shader: 減算合成
  • Sprites-Multiply.shader: 乗算合成
  • Sprites-Screen.shader: スクリーン合成
  • Sprites-Invsrc.shader: 反転ソース合成

ExpresserComponent連携

ShaderFactoryは他のVaNillaモジュールとシームレスに連携します：

[RequiredExpresserComponent(typeof(VaNilla.Renderer.BasicRenderer))]
public class ChangeShading : IExpresserComponent
{
    public ShaderResource shader { get; set; } = ShaderManager.GetDefaultShader();
    
    public void UpdateShader(ShaderResource shader__)
    {
        shader = shader__;
        renderer.UpdateRenderer();
    }
}

パフォーマンス考慮事項

  • 事前ロード: 頻繁に使用するシェーダーはゲーム開始時に事前ロード
  • シェーダー切り替え: 過度なシェーダー切り替えはパフォーマンスに影響
  • リソース管理: 使用しなくなったシェーダーは適切にUnload

拡張性

新しいブレンドモードやカスタムシェーダーを追加する場合：

  1. ShaderManager.ShaderEnumに新しい列挙値を追加
  2. 対応するシェーダーファイルをFixture/Unity/に配置
  3. プラットフォーム固有の実装をImplementation/に追加

依存関係

  • VaNilla.Renderer: BasicRendererとの連携
  • VaNilla.Texture: テクスチャリソースとの統合
  • VaNilla.Execute: ExpresserComponentシステム

技術仕様

  • 対応プラットフォーム: DxLib, Unity
  • シェーダー言語: Unity HLSL (Unity), プラットフォーム固有 (DxLib)
  • リソースフォーマット: プラットフォーム依存
  • メモリ管理: 参照カウント方式

ShaderFactoryモジュールにより、VaNillaアプリケーションで豊富な視覚効果を簡単に実現できます。 プラットフォーム間の違いを抽象化し、一貫したシェーダー管理体験を提供します。
