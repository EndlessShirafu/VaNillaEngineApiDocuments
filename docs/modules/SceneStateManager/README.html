SceneStateManager

VaNillaゲームエンジンにおけるシーン管理システムです。
有向グラフベースのシーン遷移制御、シーンスタック機能、オブジェクトライフサイクル管理を提供します。

概要

SceneStateManagerは、ゲーム内のシーン（画面状態）を管理し、シーン間の遷移を制御するためのモジュールです。
YAMLで定義された有向グラフに基づいて、許可されたシーン遷移のみを実行し、不正な遷移を防ぎます。

主な特徴

  • 有向グラフベースの遷移制御 - 許可された遷移パスのみを実行可能
  • シーンスタック機能 - Push/Pop操作によるモーダルなシーン管理
  • シーングループ - 関連するシーンをグループ化して管理
  • 自動オブジェクト管理 - シーン遷移時の自動的なオブジェクト破棄
  • シーン処理の分離 - 各シーンの処理をインターフェースで分離
  • YAML駆動のコード生成 - シーン定義をYAMLで記述し、C#コードを自動生成

使用方法

1. シーン定義（YAML）

VaNillaSettings.yamlでシーンとその遷移を定義します：

scene_state:
  InitialLoading:
    group: 0
    transit: [CircleLogo]
  
  CircleLogo:
    group: 0
    transit: [Title]
  
  Title:
    group: 1
    stack: true
    transit: [Config, RootConsole]
  
  RootConsole:
    group: 2
    stack: true
    transit: [Tutorial, Config]
  
  Tutorial:
    group: 2
    transit: [RootConsole]
  
  Config:
    group: 2
    transit: [RootConsole]

2. コード生成

YAMLからC#コードを生成：

python VaNillaEngineManager/CodeGenerator.py -s VaNillaSettings.yaml

3. シーン処理の実装

各シーンの処理をISceneStateProcessorインターフェースで実装：

[SceneStateProcess(SceneStateManager.SceneState.Title)]
public class TitleSceneProcessor : ISceneStateProcessor
{
    public void Init()
    {
        // ゲーム起動時、一度だけ実行される初期化処理
    }
    
    public void EnteredScene(in Time current_time)
    {
        // シーンに入った時の処理
        // UI表示、BGM再生など
    }
    
    public void Execute(in Time current_time)
    {
        // シーン中、毎フレーム実行される処理
        // 入力処理、画面更新など
    }
    
    public void LeavedScene(in Time current_time)
    {
        // シーンから出る時の処理
        // リソース解放、BGM停止など
    }
    
    public void Fin()
    {
        // ゲーム終了時の処理
    }
}

4. シーン遷移の実行

// 通常のシーン遷移
if (SceneStateManager.SetCurrentScene(SceneState.RootConsole))
{
    // 遷移成功
}

// シーンをスタックにプッシュ（モーダル表示）
if (SceneStateManager.PushScene(SceneState.Config))
{
    // Config画面をモーダル表示
}

// シーンをポップ（前の画面に戻る）
SceneState poppedScene = SceneStateManager.PopScene();

// 現在のシーン情報取得
SceneStateInfo info = SceneStateManager.GetCurrentSceneStateInfo();
string sceneName = info.scene_name;
SceneStateGroup group = info.scene_state_group;

シーンスタック機能

Push/Pop操作

シーンスタックを使用することで、モーダルダイアログやサブメニューの実装が容易になります：

  1. PushScene - 現在のシーンをスタックに保存し、新しいシーンへ遷移
  2. PopScene - スタックから前のシーンを取り出して復帰

スタック可能条件

YAMLのstackフラグで、そのシーンから他のシーンへの遷移がスタック可能かを制御：

  • stack: true: このシーンから他のシーンへPushScene()で遷移可能（モーダル表示）
  • stack未指定またはfalse: このシーンからの遷移はSetCurrentScene()のみ可能（通常遷移）

例：Titleにstack: trueが設定されている場合、TitleからConfigやRootConsoleへPushScene()で遷移できます。

シーングループ

グループの概念

関連するシーンを同じグループに配置することで、まとめて管理できます：

  • GROUP_0: ローディング・ロゴ表示
  • GROUP_1: メインメニュー系
  • GROUP_2: ゲームプレイ系

グループ遷移時の動作

異なるグループへ遷移する際は、シーンスタックが自動的にクリアされます。
これにより、メインメニューからゲーム画面への遷移時など、スタックをリセットしたい場面で便利です。

オブジェクトライフサイクル管理

シーン遷移時のオブジェクト破棄

シーン遷移時、デフォルトではすべてのObjectが破棄されます：

// SceneStateProcessor内で自動実行
ObjectExecutor.DestroyObjectOnSceneChanged();

オブジェクトの永続化

特定のObjectをシーン遷移後も保持したい場合、DontDestroyOnSceneChangedAttributeを使用：

[DontDestroyOnSceneChanged]
public class PersistentManager : VaNilla.Object
{
    // このオブジェクトはシーン遷移時も破棄されない
    // BGMマネージャー、セーブデータ管理など
}

シーン変更の検出

SceneChangedDetectorとの連携

SceneState pastScene, currentScene;
if (SceneChangedDetector.IsSceneChaged(out pastScene, out currentScene))
{
    // シーンが変更された
}

// または
if (SceneStateManager.GetSceneChanged())
{
    // シーン変更フラグをチェック
    SceneStateManager.ClearSceneChanged(); // フラグをクリア
}

実行フロー

初期化時

  1. SceneStateProcessor.Initialize() が呼ばれる
  2. アセンブリからSceneStateProcessAttributeが付いたクラスを収集
  3. 各シーンプロセッサのInit()を実行

毎フレーム実行

  1. SceneStateProcessor.Execute() が呼ばれる
  2. シーン変更を検出
  3. 変更があれば：
    • 前シーンのLeavedScene()を実行
    • オブジェクトの破棄処理
    • GCの明示実行
    • 新シーンのEnteredScene()を実行
  4. 現在シーンのExecute()を実行

ベストプラクティス

1. シーン設計の指針

  • 関連する画面は同じグループに配置
  • モーダル表示が必要な画面はcan_stack: trueに設定
  • 遷移パスは必要最小限に制限

2. リソース管理

  • 重いリソースはEnteredSceneで読み込み、LeavedSceneで解放
  • 共通リソースはDontDestroyOnSceneChanged付きマネージャーで管理

3. パフォーマンス最適化

  • シーン遷移時のGC実行を考慮し、大量のオブジェクト生成は避ける
  • 頻繁に切り替わるシーンはスタック機能を活用

注意事項

  • コード生成必須 - SceneStateManager.csは自動生成されるため、直接編集しない
  • 遷移制約の遵守 - YAMLで定義されていない遷移は実行できない
  • シングルスレッド - VaNillaエンジン全体がシングルスレッドモデルのため、非同期遷移は不可

関連モジュール

  • ObjectExecutor - オブジェクト実行管理
  • Object - ゲームオブジェクトシステム
  • SceneChangedDetector - シーン変更検出
  • VaNillaEngineManager - コード生成ツール

サンプルコード

タイトル画面からゲーム画面への遷移

[SceneStateProcess(SceneStateManager.SceneState.Title)]
public class TitleScene : ISceneStateProcessor
{
    public void Execute(in Time current_time)
    {
        if (KeyboardInput.GetKeyDown(KeyCode.Space))
        {
            // スペースキーでゲーム開始
            SceneStateManager.SetCurrentScene(SceneState.RootConsole);
        }
        else if (KeyboardInput.GetKeyDown(KeyCode.C))
        {
            // Cキーで設定画面をモーダル表示
            SceneStateManager.PushScene(SceneState.Config);
        }
    }
    // 他のメソッドは省略
}

設定画面からの復帰

[SceneStateProcess(SceneStateManager.SceneState.Config)]
public class ConfigScene : ISceneStateProcessor
{
    public void Execute(in Time current_time)
    {
        if (KeyboardInput.GetKeyDown(KeyCode.Escape))
        {
            // ESCキーで前の画面に戻る
            SceneStateManager.PopScene();
        }
    }
    // 他のメソッドは省略
}
