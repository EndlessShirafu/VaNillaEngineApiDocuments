SoundManager

音響リソースの管理と再生を行うモジュールです。効果音やBGMの読み込み、再生、音量制御などを統一されたAPIで提供します。

概要

SoundManagerは、VaNillaエンジンにおける音響システムの中核を担います。カテゴリベースのリソース管理、高度な再生制御、プラットフォーム抽象化を通じて、音響機能を提供します。

APIリファレンス

詳細なAPIドキュメントはVaNilla SoundManager APIリファレンスをご覧ください。

主要機能

リソース管理

  • カテゴリ別音響リソース管理（music、sound_effect等）
  • 非同期読み込み対応
  • 一括読み込み・解放機能
  • YAMLベースの設定ファイルからの自動生成

音響再生

  • 3つの再生タイプをサポート
    • OneShot: 一回再生
    • Loop: 単純ループ再生
    • RegionLoop: 複数リージョンを持つ動的音楽システム
  • フェードイン・フェードアウト効果
  • パン（L/R）制御
  • サンプル精度の位置制御

音量制御

  • マスター音量による全体制御
  • カテゴリ別音量制御
  • 個別音響の音量設定
  • 階層的音量管理システム

アーキテクチャ

インターフェース層（Interface/）

  • SoundResourceManager: 音響リソース管理の中核クラス
  • SoundResourceLoader: リソース読み込み・解放ユーティリティ
  • Utils: 高レベル再生制御API
  • SoundResourceType: 音響リソース種別列挙型（自動生成）

実装層（Implementation/）

  • DxLib: DxLibDotNetを使用した完全実装
  • Unity: Unity向け実装（開発中）

データ構造

  • SoundInfo: 音響メタデータ（ファイルパス、再生設定等）
  • RegionLoopInfo: リージョンループ情報（開始・終了・ループポイント）
  • SoundResource: プラットフォーム固有リソースハンドル

使用方法

基本的な使用例

// カテゴリ別読み込み
SoundResourceManager.LoadSoundResource(SoundCategory.sound_effect);

// 音響再生
Utils.Play(SoundResourceType.TestSE);

// 音量制御
Utils.SetVolume(SoundResourceType.TestMusic, 0.5f);

// ループ音楽の制御
Utils.TransitToNextRegion(SoundResourceType.TestMusic, 2);

非同期読み込み

// 非同期読み込み開始
SoundResourceManager.LoadSoundResourceAsync(SoundCategory.music);

// 読み込み完了待機
while (!SoundResourceManager.IsLoaded(SoundCategory.music))
{
    await Task.Delay(16);
}

フェード効果

// フェードイン再生
Utils.FadeIn(SoundResourceType.BGM, 2.0f, 1.0f);

// フェードアウト停止
Utils.FadeOut(SoundResourceType.BGM, 1.5f);

設定ファイル

音響リソースはYAMLファイルで設定し、コード生成によりSoundResourceType列挙型が自動生成されます。

# VaNillaSettings.yaml
sound_pool:
  # BGMのループ再生
  BGM_Title:
    path: "audio/bgm_title.ogg"
    type: "loop"
    loop_samples: 44100
    volume: 0.8
    category: "bgm"
  
  # 効果音の単発再生
  SE_Shot:
    path: "audio/se_shot.wav"
    type: "oneshot"
    volume: 1.0
    category: "sound_effect"
  
  # 音声の領域ループ再生
  Voice_Character:
    path: "audio/voice_character.ogg"
    type: "region_loop"
    volume: 0.9
    category: "voice"
    region:
      greeting:
        index: 0
        start: 0
        loop_start: 1000
        end: 3000
      goodbye:
        index: 1
        start: 4000
        loop_start: 5000
        end: 7000

依存関係

  • AssetManipulator: ファイルパス解決
  • DebugLogger: ログ出力
  • Task: 非同期処理
  • TimeManager: フェード効果のタイミング制御
  • SupremeArbiter: FPS値の取得

初期化

SoundManagerはSupremeArbiterによって自動的に初期化されます。手動初期化が必要な場合：

SoundResourceManager.Initialize();

注意事項

  • Unity実装は現在開発中のため、Unity環境では一部機能が制限されます
  • リージョンループ機能は高度な音楽システム向けであり、複雑な設定が必要です
  • 非同期読み込み中の再生要求は無視される場合があります
  • メモリ効率のため、使用しないカテゴリは適切に解放してください

パフォーマンス考慮事項

  • カテゴリ別読み込みによりメモリ使用量を最適化
  • 非同期読み込みによりフレームレート維持
  • リソース解放による動的メモリ管理
  • サンプル精度制御による高精度音響システム
