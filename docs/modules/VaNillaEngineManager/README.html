VaNillaEngineManager

VaNillaEngineのプロジェクト全体を扱いやすくするためのツールなどをまとめるディレクトリです。

プロジェクト設定 yaml について

VaNillaでは、極力「事前に静的に確定するリソース」については、 事前に生成しておいてオーバーヘッドを減らそうという思想があります。 それを実現するために、ゲーム中で利用するリソースなどの情報は プロジェクト設定 yaml (VaNillaSettings.yaml) に記載しておきます。 「静的に確定するリソース」の例は下記です。

  • ゲーム内で使うテクスチャ群
  • ゲーム内で使う音リソース群
  • 画面遷移設定
  • キーバインド設定
  • etcetc...

プロジェクト設定 yaml (VaNillaSettings.yaml) は、各ゲームの要件に応じて適宜修正する必要があります。

SharedProjectGenerator

プロジェクト設定 yaml (VaNillaSettings.yaml)を元にして、 VaNillaを構成する共有プロジェクト(.projitems)を構築します。 出力先ファイルは、 VaNillaShared/VaNillaShared.projitems です。 以下のように呼び出します。

SharedProjectGenerator.exe -s [VaNillaSettingsフォルダのパス]

VaNillaに含めてビルドするソースコードを追加/削除したい場合には、本ツールを利用してください。 逆に、上記以外の用途で利用することはありません。

CodeGenerator

プロジェクト設定 yaml (VaNillaSettings.yaml)を元にして、ソースコードを生成します。 以下のように呼び出します。

CodeGenerator.exe -s [VaNillaSettingsフォルダのパス]

SharedProjectGenerator と CodeGenerator のビルドについて

SharedProjectGenerator.exe と CodeGenerator.exe は、 それぞれ SharedProjectGenerator.py と CodeGenerator.py を PyInstaller で exe 化したファイルです。 SharedProjectGenerator.py と CodeGenerator.py の中身の処理を修正した場合には、 create_exe.bat を実行することで各 exe を再生成できます。

VaNillaShared

VaNilla の Interface 層と Implementation 層を共有プロジェクト化し、まとめたディレクトリです。 VaNillaを利用する側の .csproj (以下アプリプロジェクト)から、下記のプロジェクトを 共有プロジェクト参照して使うことを想定しています。

  • VaNillaShared/VaNillaShared.shproj

VaNilla を含めたアプリプロジェクトのビルドには、 CodeGenerator と SharedProjectGenerator で事前に共有プロジェクト設定とソースコードを生成しておく必要があります。 （上記を実施していない場合、ObjectFactory などでビルドエラーとなります）

VaNillaEngineManagerApp

VaNillaのUnityへのインストールをGUI経由で実行できます。 性質上、Implementation に Unity を使うケースでしか利用しません。

config.ini という名前のファイルで、 1行目にdata_version = 2、 2行目にVaNillaをインストールしたいUnityプロジェクトのパス を書いておくと起動時に勝手に config.ini から読んでくれます。(.gitignore指定済み)

インストール先のUnityプロジェクトパスを入れたうえで InstallAll ボタンを押すとVaNillaのインストール処理を行えます。 インストールは具体的にはシンボリックリンクの作成や初期設定等が行われる。

SymbolicLinkGeneratorForUnity

Unity用にシンボリックリンク作るやつです。
実行にはPython3.6以上をコマンドプロンプト上から呼び出せる環境が必要です。

install_for_unity_cmd_helper.bat

任意のUnityプロジェクトへVaNillaモジュールのシンボリックリンクを作成するツールです。

miscellaneous_setup.py

その他雑多な初期設定などを行うためのスクリプトです。
リソース関連のセットアップなどのために呼び出し必須。
install_all_for_unity_project.py の呼び出し後に miscellaneous_setup.py も合わせて呼び出すこと。

使い方

  1. cmdプロンプト開く
  2. install_for_unity_cmd_helper.bat を実行
  3. 管理者権限で別cmdプロンプトが開く
  4. 開いたcmdプロンプトで下記を実行 python install_all_for_unity_project.py -l [Unityプロジェクト直下へのパス]
    ※[Unityプロジェクト直下へのパス]はAssetsフォルダなどがある地点を指定
  5. 実行すると指定したUnityプロジェクトへインストールされるはず

ファイル構成

  • install_for_unity_cmd_helper.bat ... 管理者権限でcmd開くためのbat
  • install_all_for_unity_project.bat ... pythonスクリプトの実行bat 正直必要ではない
  • install_all_for_unity_project.py ... 本体のスクリプト

メンテ

install_all_for_unity_project.py 内にインストールするディレクトリのリストがある 手動追従するのめんどくさくなってくる気がするのでそのうちいい感じに変える
