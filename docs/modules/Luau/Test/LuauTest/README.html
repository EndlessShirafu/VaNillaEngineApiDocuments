Luau Test Project

VaNillaのLuauモジュールの包括的なテストスイートです。

テスト概要

このテストプロジェクトは以下のコンポーネントをテストします：

1. VaNillaLuauAttributeTest

  • VaNillaLuauAttribute の基本機能
  • プロパティの設定と取得
  • デフォルト値の検証
  • 属性の継承関係

2. LuauAPIAttributeTest

  • LuauAPIAttribute の基本機能
  • API マーキング機能
  • パラメータ制約の管理
  • 読み取り専用設定

3. LuauExecutorTest

  • Luau実行エンジンの初期化・シャットダウン
  • スクリプトの登録・実行・管理
  • エラーハンドリング
  • 非同期実行

4. LuauTypeConverterTest

  • C# ⟷ Luau 型変換システム
  • 基本型、配列型、ジェネリック型の変換
  • カスタム型ハンドラー
  • VaNillaコア型定義の生成

5. LuauIntegrationTest

  • 全コンポーネントの統合テスト
  • ライフサイクル管理
  • 複数オブジェクトの処理
  • パフォーマンス監視（モック）

6. LuauExpresserComponentTest

  • IExpresserComponent への VaNillaLuau 属性適用テスト
  • LuauAPI 属性によるメンバー公開テスト
  • 複数コンポーネントの設定テスト

7. LuauScriptExecutionTest

  • 実際のLuauスクリプトファイルの実行テスト
  • 正常系のスクリプト実行検証
  • プロパティアクセスとメソッド呼び出しのテスト
  • エラーハンドリングの実動作テスト

テストの実行方法

1. コマンドラインから実行

# テストプロジェクトのディレクトリに移動
cd VaNilla/Luau/Test/LuauTest/

# すべてのテストを実行
dotnet test

# 詳細な出力でテスト実行
dotnet test --verbosity normal

# 特定のテストクラスのみ実行
dotnet test --filter "FullyQualifiedName~LuauExecutorTest"

# 特定のテストメソッドのみ実行
dotnet test --filter "FullyQualifiedName~LuauExecutorTest.Initialize_DoesNotThrow"

2. Visual Studio から実行

  1. LuauTest.sln をVisual Studioで開く
  2. テストエクスプローラーを表示（テスト → テストエクスプローラー）
  3. 「すべて実行」または個別のテストを実行

重要な注意事項

実装状況について

現在のLuauモジュールは 実際のLuau APIを使用した実装 になりました：

  • Luau.dll を使用した実際のスクリプト実行
  • LuauCompiler によるバイトコードコンパイル
  • LuauState による実行コンテキスト管理
  • 実際のLuauValue型変換システム

Luau.Native.dll について

一部のテストはLuau.Native.dllを必要とします。このファイルが見つからない場合：

  • LuauScriptExecutionTestの正常系テストはスキップされます
  • エラーハンドリングのテストは実行されます

テストの想定

  1. 非侵襲的設計の検証 - 既存コードに影響しないことを確認
  2. API契約の検証 - インターフェースが正しく動作することを確認
  3. エラーハンドリングの検証 - 適切な例外処理を確認
  4. 統合動作の検証 - 全コンポーネントが連携して動作することを確認

テスト結果の解釈

成功時

すべてのテストが通る場合、Luauモジュールの基本設計が正しく動作しています。

失敗時

  1. コンパイルエラー - 依存関係の問題
  2. Null Reference Exception - 初期化順序の問題
  3. Argument Exception - パラメータ検証の問題
  4. Invalid Operation Exception - ライフサイクル管理の問題

今後の拡張

追加実装予定のテスト

  1. 複雑なLuauスクリプト実行テスト - より高度なロジックの検証
  2. パフォーマンスベンチマークテスト - 実行速度の測定
  3. メモリリーク検出テスト - 長時間実行時の安定性
  4. ホットリロード機能テスト - 動的スクリプト更新
  5. デバッグ機能テスト - ブレークポイントとステップ実行
  6. 並列実行テスト - 複数スクリプトの同時実行
  7. エラーリカバリーテスト - エラー後の状態回復
