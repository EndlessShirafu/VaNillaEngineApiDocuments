WinRtGlobalizationBridge

このモジュールは、Windows Runtime (WinRT) の Windows.System.UserProfile.GlobalizationPreferences API をラップし、通常の .NET プロジェクトから WinRT 依存なしで利用できるようにするブリッジライブラリです。

目的

VaNilla の LocalizationUtil.cs で GlobalizationPreferences.Languages を使用する際、通常の C# プロジェクトではエラーが発生します。このブリッジライブラリにより、WinRT への依存部分を別 DLL に分離し、メインプロジェクトを WinRT 非依存に保ちます。

アーキテクチャ

1. ラッパーDLL (WinRtGlobalizationBridge.dll)

  • ターゲットフレームワーク: net9.0-windows10.0.19041.0
  • 依存関係: Microsoft.Windows.SDK.Contracts
  • WinRT API: Windows.System.UserProfile.GlobalizationPreferences を直接使用

2. メインプロジェクト (VaNilla)

  • ターゲットフレームワーク: net9.0 (WinRT 非依存)
  • DLL 呼び出し: リフレクションによる動的呼び出し
  • フォールバック: DLL が利用できない場合は "en-US" を返却

使用方法

WinRtGlobalizationBridge.dll のビルド

cd VaNilla/WinRtGlobalizationBridge
dotnet build --configuration Release

メインプロジェクトでの使用

LocalizationUtil.cs は自動的にラッパーDLLを呼び出します：

// OSの優先言語を取得
string preferredLanguage = VaNilla.Localization.Util.GetPreferredLanguage();

API 仕様

GlobalizationHelper クラス

public static class GlobalizationHelper
{
    /// <summary>
    /// OSの優先言語設定一覧を取得します
    /// </summary>
    /// <returns>言語タグのリスト（例: "ja-JP", "en-US"）</returns>
    public static IReadOnlyList<string> GetPreferredLanguages();
}

エラーハンドリング

  • WinRT DLL が見つからない場合: "en-US" をデフォルト値として返却
  • WinRT API 呼び出し失敗: 例外をキャッチしてフォールバック値を返却
  • リフレクション失敗: 例外をキャッチしてフォールバック値を返却

配置要件

  • WinRtGlobalizationBridge.dll をメインアプリケーションと同じディレクトリに配置
  • Windows 10 version 1903 (build 19041) 以降が必要

テスト

DLL の動作をテストする場合：

var languages = WinRtGlobalizationBridge.GlobalizationHelper.GetPreferredLanguages();
foreach (var lang in languages)
{
    Console.WriteLine(lang);
}

依存関係

  • Microsoft.Windows.SDK.Contracts (10.0.22621.30401)
  • .NET 9.0 targeting Windows 10.0.19041.0

注意事項

  • このライブラリは Windows 環境でのみ動作します
  • WinRT API が利用できない環境では自動的にフォールバック値を返却します
  • リフレクション呼び出しのため、パフォーマンスが重要な箇所では注意が必要です
