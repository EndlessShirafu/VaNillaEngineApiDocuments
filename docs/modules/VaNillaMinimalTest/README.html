VaNilla Minimal Test

VaNillaエンジンの最小構成で動作するテストプロジェクトです。

概要

このプロジェクトは、VaNillaエンジンを理解するための最小限の構成で作成されています。複雑な機能を省き、エンジンの基本的な動作を確認することができます。

目的

  • VaNillaエンジンの基本的なアーキテクチャの理解
  • 最小限のモジュール構成での動作確認
  • 新規開発者向けの学習用プロジェクト
  • エンジンの軽量性の実証

含まれるモジュール

最小限の動作に必要な以下のモジュールのみを使用：

必須コアモジュール

  • SupremeArbiter - エンジンの初期化・実行・終了制御
  • Object - オブジェクトシステムの基底クラス
  • ExpresserComponent - コンポーネントシステム
  • ObjectExecutor - オブジェクトとコンポーネントの実行管理

基本機能モジュール

  • DebugLogger - デバッグ出力
  • Transform - 座標・回転・拡縮の基本変換

ビルドと実行

前提条件

  • .NET 9 SDK
  • Python 3.x（コード生成用）
  • Windows環境（DxLib使用のため）

現在の制限事項

⚠️ 重要: 現在のMinimalTestプロジェクトは、VaNillaエンジンのアーキテクチャ上の制約により、完全に独立した「最小構成」での動作ができません。VaNillaEngineManager/VaNillaShared.projitemsを通じて全モジュールが参照されるため、実際にはすべてのモジュールの生成されたファイルが必要になります。

真の最小構成を実現するためには、VaNillaShared.projitemsから不要なモジュールを除外するか、独立したプロジェクト構造が必要です。

手順

  1. コード生成

    cd VaNillaMinimalTest/DxLib
./update_code.bat

  2. ビルドと実行

    dotnet build project/MinimalTest.csproj
dotnet run --project project/MinimalTest.csproj

または：

./install_and_run_test.bat

通常のVaNillaTestProjectとの違い

項目 VaNillaTestProject VaNillaMinimalTest
モジュール数 30+ 6
テストケース 複数の機能テスト 基本動作のみ
複雑度 高（全機能） 低（最小限）
学習コスト 高 低
ビルド時間 長 短

拡張方法

このMinimalTestをベースに、段階的に機能を追加できます：

  1. 描画機能 - Renderer、TextureResourceManagerを追加
  2. 入力処理 - KeyboardInput、GamePadInputを追加
  3. サウンド - SoundManagerを追加
  4. 衝突判定 - Collisionを追加

各ステップの詳細はDocumentation/Architecture.mdを参照してください。
