WindowSettingUpdater

画面サイズや画面解像度などの情報を反映するためのモジュールです。

概要

WindowSettingUpdaterは、VaNillaエンジンにおける画面設定の変更をプラットフォーム固有の実装に反映するための中間モジュールです。ScreenResolutionManagerとWindowSizeManagerで管理される論理的な設定を、DxLibやUnityの実際の画面設定に同期させる役割を担います。

主要機能

  • 自動設定同期: 画面解像度、ウィンドウサイズ、フレームレートの変更を自動検出
  • 差分更新: 変更が検出された場合のみプラットフォーム固有の更新処理を実行
  • 統一API: DxLibとUnityの両プラットフォームで同一のインターフェースを提供
  • 定期実行対応: ゲームループ内での定期的な呼び出しに最適化

基本的な使用方法

using VaNilla.Window;

// ゲームループ内で定期的に呼び出す
void Update()
{
    // 画面設定の変更を検出し、必要に応じてプラットフォームに反映
    WindowSettingUpdater.UpdateSetting();
}

動作の仕組み

WindowSettingUpdaterは以下の流れで動作します：

  1. 現在の設定取得: ScreenResolutionManager、WindowSizeManager、FpsCalculatorから現在の設定を取得
  2. 差分検出: 前回の設定と比較し、変更があったかを確認
  3. プラットフォーム反映: 変更が検出された場合、ネイティブ実装を通じてプラットフォームに反映

連携モジュール

  • ScreenResolutionManager: 論理解像度の管理
  • WindowSizeManager: ウィンドウサイズとフルスクリーン設定の管理
  • FpsCalculator: フレームレート設定の管理

API Reference

詳細なAPIドキュメントについては、VaNillaのAPIリファレンスを参照してください。

WindowSettingUpdater

画面設定の変更をプラットフォーム固有の実装に反映するクラスです。

静的メソッド

  • UpdateSetting() - 画面解像度やウィンドウサイズ、リフレッシュレートに関する設定値をプラットフォームに反映

使用上の注意

  • 本モジュールは画面描画更新と同程度の粒度で定期的に実行されることを想定しています
  • 変更検出は差分ベースのため、頻繁な呼び出しでもパフォーマンスへの影響は最小限です
  • プラットフォーム固有の実装に依存するため、対応プラットフォームでのみ動作します
