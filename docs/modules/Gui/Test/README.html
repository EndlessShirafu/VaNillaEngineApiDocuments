VaNilla GUI モジュールテスト

このディレクトリにはxUnitフレームワークを使用したVaNilla GUIモジュールのユニットテストが含まれています。

テスト構造

テストファイル

  • ControlTests.cs - 基底Controlクラスの機能テスト
  • ButtonTests.cs - Buttonコントロールのテスト
  • FormTests.cs - Formコントロールのテスト
  • TextBoxTests.cs - TextBoxコントロールのテスト
  • LabelTests.cs - Labelコントロールのテスト
  • ListControlTests.cs - ListControlのテスト
  • ScrollBarTests.cs - ScrollBarとVScrollBarのテスト
  • HScrollBarTests.cs - HScrollBarのテスト
  • MessageBoxTests.cs - MessageBox静的メソッドのテスト
  • GraphicalUITests.cs - GraphicalUIコンテナのテスト
  • XmlGUICreatorTests.cs - XMLベースのGUI作成のテスト

テストカバレッジ

各テストファイルは以下をカバーしています：

  • コンストラクタの動作とデフォルト値
  • プロパティのゲッターとセッター
  • 継承関係
  • 基本機能とメソッド呼び出し
  • 状態管理（有効/無効、表示/非表示）
  • Renderメソッドの動作

テストの実行

前提条件

  • .NET 9.0 SDK以降
  • xUnitテストランナー

コマンド

# テストプロジェクトのビルド
dotnet build GuiTest/GuiTest.csproj

# すべてのテストを実行
dotnet test GuiTest/GuiTest.csproj

# 詳細出力付きでテストを実行
dotnet test GuiTest/GuiTest.csproj --logger "console;verbosity=detailed"

# 特定のテストクラスを実行
dotnet test GuiTest/GuiTest.csproj --filter "ClassName=ControlTests"

# コードカバレッジレポートを生成
dotnet test GuiTest/GuiTest.csproj --collect:"XPlat Code Coverage"

依存関係

テストプロジェクトには以下の依存関係が含まれています：

  • xUnit - テストフレームワーク
  • Moq - ユニットテスト用のモックフレームワーク
  • Coverlet - コードカバレッジ収集
  • Microsoft.NET.Test.Sdk - .NETテストSDK

テストプロジェクトの設定

テストプロジェクトは以下を含む最新のSDKスタイルのプロジェクト形式を使用しています：

  • ターゲットフレームワーク： .NET 9.0
  • テスト依存関係のパッケージ参照
  • GUIインターフェースファイルへの直接ファイル参照
  • 他のVaNillaモジュールからのリンクされた依存関係

注意事項

  • テストはパブリックAPIの動作に焦点を当て、内部実装の詳細はテストしません
  • 一部のテストは、直接アクセスができない場合にプロパティ/メソッドの存在を確認するためにリフレクションを使用します
  • MessageBoxテストは静的メソッドのシグネチャと列挙値を検証します
  • XmlGUICreatorテストは適切なクリーンアップを含むファイルI/O操作を含みます
  • すべてのテストは独立して設計されており、任意の順序で実行できます

将来の拡張
